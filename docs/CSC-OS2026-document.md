# CSC-OS2026 document


### 总体要求
- 除特别要求、规定和禁止的事项外，可以自行决定比赛采用的操作系统、体系结构、内部功能规范、具体实现细节等。
- 若在作品中复制了著作权非本队原创的源代码和文档，必须在作品所复制的源代码和文档位置处明确写出来源，并在作品的设计方案文档中进一步总体说明整个作品所复制部分的来源、使用目的等内容。若作品复制的源代码和文档遵循特定开源协议（ 参见https://opensource.org/license ），参赛队必须严格遵守该协议的各项规定，确保源代码和文档使用的合法。
- 参赛作品的源代码需遵循开源协议，符合GPL协议\Apache协议\BSD协议\木兰协议中至少一种。
- 参赛队应及时将作品的逐步改进版本分多次提交至指定的托管平台，展示参赛队完整的开发过程，以便进行自动化和人工评测。每次提交间包含详细的注释说明（比如新增功能和修复关键Bug等），禁止无注释提交，鼓励频繁提交并全程详细真实记录开发迭代过程。


### 说明
- 参赛队应运用操作系统、编译技术、计算机体系结构等知识，设计并实现一个体现目标平台优化能力的操作系统内核。
- 结合语言和硬件特性，设计兼顾正确性与效率的系统，保障评测用例的运行。
- 可参考往届作品或开源实现，但须在源码与文档中明确标注来源，并说明基础版本与自身改进。
- 不要求从零实现内核，可基于开源参考实现或模块开发，但要突出自主完成的增量贡献。
- 参考或复制他人实现时，须遵守比赛要求，确保源码与文档合法合规。
- 内核须支持 RISC-V64 与 LoongArch64；初赛仅在 QEMU 模拟环境中测试。

### 提交内容

提交的内容中必须包括操作系统内核的完整工程文件和操作系统设计方案文档。
- 操作系统内核的完整工程文件。必须包含全部操作系统源代码（有源代码注释，中英文均可，著作权非本队的源代码和文档必须有来源说明等注释）、辅助编译的文件、建立可正确编译操作系统内核的编译环境的Docker文件等，并在自动测试平台中至少有一次完整通过功能与性能测试的记录和有效成绩。
- 操作系统设计方案等开发相关文档。文档内容包括但不限于：开发操作系统的设计思路、实现重点、开发过程中遇到的问题和解决方法、在本队作品源码和文档中存在的非本队来源说明、本技术方案1.1节第5点所描述的要求等。

### 硬件平台

操作系统设计赛指定的“内核实现”赛道的硬件设备有两种。发放的虚拟硬件平台为QEMU for RISC-V和LoongArch；发放的物理硬件平台为基于RISC-V 的VisionFive 2星光二代开发板和基于LoongArch的2K1000LA 开发板。初赛环境基于虚拟硬件平台，决赛环境基于虚拟硬件平台和物理硬件平台。

物理硬件平台可由参赛队在报名完成后向秘书处申请，由秘书处统一发放。

1. VisionFive 2 星光二代板主要参数如下：
- CPU: 64 位SiFive FU740 SoC，集成4 个1.5GHz U74-MC 内核＋1 个S7 嵌入式内核
- 内存：2~8GB LPDDR4 RAM
- 板载32MB SPI 闪存芯片
- HDMI 2.0
- USB 2.0/3.0
- 千兆网卡

2. 广东龙芯2K1000 星云板主要参数如下：
- CPU: 龙芯2K1000LA 处理器
- 内存：2~8GB DDR3 RAM
- 2 个USB 2.0 接口
- 1 个OTG 接口
- 1 个标准HDMI 接口
- 2 个千兆网口
- PCIe X1 扩展接口
